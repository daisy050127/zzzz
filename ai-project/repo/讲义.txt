1.2 核心概念速查表
概念	英文	简要说明
系统提示词	System Prompt	定义AI角色、行为边界、输出格式的顶层指令
用户提示词	User Prompt	用户向AI提出的具体任务或问题
令牌	Token	LLM处理文本的最小单位（约0.75个英文单词）
上下文	Context	包含系统提示、历史消息、检索信息的全部输入
检索增强生成	RAG	从外部知识库检索相关内容，辅助LLM回答
函数调用	Function Call	LLM请求执行外部函数的能力
技能	Skill	结构化的专业能力定义文件（SKILL.md）
模型上下文协议	MCP	AI与外部工具交互的标准化协议
智能体	Agent	能自主规划、调用工具、执行多步任务的系统
第2章 基础提示词技巧
2.1 角色身份定义
核心原则：为AI设定明确的角色身份，能极大提升回答的专业性和一致性。
markdown
# 模板你是一位{专业领域}专家，拥有{经验年限}年{行业}经验。用户是一位{用户角色}，其需求是{需求描述}。请以{语气风格}回答，输出格式为{指定格式}。
案例：双AI辩论
markdown
你是一位反方辩手，擅长逻辑拆解。用户是一位正方辩手，需要构建说服性论点。请按照“对方观点→逻辑漏洞→反驳论点→总结”的结构回复。
2.2 回答模板约束
强制AI按照预设结构输出，便于下游解析。
markdown
请按以下结构回答：### 问题重述[用一句话复述用户问题]### 核心分析- 点1（不超过50字）- 点2（不超过50字）- 点3（不超过50字）### 结论与建议[1-2句话总结]
2.3 上下文手工组装与压缩
手工组装：将系统提示、历史对话、检索文档拼接成完整上下文。
压缩技巧（摘要法）：
对前70%的对话历史生成摘要
保留最后30%的原文
使用LLM执行压缩：请将以下对话历史压缩为300字摘要，保留关键决策和未解决问题
2.4 实战案例：创作100万字小说
流程：
定义AI角色（小说家）和用户角色（编辑）
双AI辩论确定故事框架和章节结构
AI主动询问细节（人物设定、世界观）
循环创作：每章输出后，手动压缩上下文（保留前情摘要+最近一章全文）
保存所有章节到本地文件
第3章 中级提示词技巧

第5章 从单次调用到流式对话
5.1 最小调用示例（Python标准库）
python
import os, json, time, urllib.requestdef call_llm(messages):    start = time.time()    data = json.dumps({        "model": os.getenv("MODEL"),        "messages": messages    }).encode()    req = urllib.request.Request(        f"{os.getenv('BASE_URL')}/chat/completions",        data=data,        headers={"Content-Type": "application/json"}    )    with urllib.request.urlopen(req) as resp:        result = json.loads(resp.read())        tokens = result['usage']['total_tokens']        elapsed = time.time() - start        return result['choices'][0]['message']['content'], tokens, elapsed

9.2 调用AnythingLLM API（使用curl）
python
def anythingllm_query(query: str) -> str:    import subprocess, os, json    slug = os.getenv("ANYTHINGLLM_WORKSPACE_SLUG")    key = os.getenv("ANYTHINGLLM_API_KEY")    cmd = [        'curl', '-X', 'POST',        f'http://localhost:3001/api/v1/workspace/{slug}/chat',        '-H', f'Authorization: Bearer {key}',        '-H', 'Content-Type: application/json',        '-d', json.dumps({"message": query})    ]    result = subprocess.run(cmd, capture_output=True, text=True)    return result.stdout

第七篇 链式工具调用（Agent核心）
第11章 链式调用原理
定义：前一个工具的输出作为后一个工具的输入参数，LLM根据中间结果自主决定下一步操作，直到完成复杂任务。
第12章 核心组件实现
12.1 链式上下文管理器
python
class ChainedCallContext:    def __init__(self, max_iterations=10):        self.steps = []          # 记录每步调用及结果        self.variables = {}      # 存储中间变量        self.max_iterations = max_iterations    def add_step(self, tool_name, arguments, result):        self.steps.append({            "step": len(self.steps)+1,            "tool": tool_name,            "args": arguments,            "result": result        })
12.2 LLM决策协议（JSON格式）
完成任务：
json
{"done": true, "answer": "最终回答内容"}
继续调用：
json
{"done": false, "tool_call": {"name": "工具名称", "arguments": {"参数名": "参数值"}}}
12.3 执行循环
python
def execute_chained_tool_call(user_request):    context = ChainedCallContext()    messages = [{"role": "system", "content": system_prompt}]        for _ in range(context.max_iterations):        # 构建分析提示词（包含用户请求和已执行步骤）        analysis_prompt = build_analysis_prompt(user_request, context.steps)        messages.append({"role": "user", "content": analysis_prompt})                response = call_llm(messages)  # 要求返回JSON        decision = parse_json(response)  # 支持markdown代码块                if decision.get("done"):            return decision["answer"]                # 执行工具调用        tool_name = decision["tool_call"]["name"]        args = decision["tool_call"]["arguments"]        result = execute_tool(tool_name, args)        context.add_step(tool_name, args, result)                # 将结果加入消息历史        messages.append({"role": "assistant", "content": json.dumps(decision)})        messages.append({"role": "user", "content": f"工具返回：{result}"})        return "达到最大迭代次数，任务未完成"
